## The boundary I got wrong this week

My first version of the Rep 2 before/after test reused the diagnoser's own
system prompt (which explicitly tells it "don't guess, search instead") for
the "tools disabled" run — so with no tools available it just said "I'd
need to search," never actually hallucinating, which silently invalidated
the test; the fix was a plain, neutral system prompt for the "before" call
so the model was actually forced to commit to an answer from training data
alone, the same failure Rep 3 later hit again as `tool_use_failed` 400s
from Groq itself, handled with a bounded retry in `agent.py` and
`threshold_test.py` rather than left to crash the run.
